# 更新说明：本体规则层与规则中心

## 概要

本次更新将 **OWL/RDF 本体** 以 **编译后的 JSON 规则包** 形式接入工作台，用于候选校验与入未验证库前的策略控制，并新增 **规则中心** 页面用于浏览当前规则内容。

## 主要能力

### 本体规则

- **规则包**：默认路径 `artifacts/ontology/ruleset.json`（可通过 `pipeline.ontology_rules.path` 配置）。
- **离线转换**：`scripts/tools/convert_owl_to_ruleset.py`（依赖 `rdflib`）。
- **工作台导入**：配置页支持上传 OWL/RDF/Turtle，服务端编译并写入配置路径后自动重载。
- **运行时**：`pipeline.ontology_rules`（启用开关、`stage_policy` hard/warn、`issue_severity` 等）。
- **校验**：`ValidationService` 对 region / circuit / connection 候选聚合本体规则结果。
- **入库门禁**：`stage_*_to_unverified` 在 `hard` 策略下拦截严重问题；命中信息写入 `review_note.ontology_check`。

### 规则中心（UI）

- 新页签 **「规则中心」**：展示 `termMap`、`parentRules`、`classRules`、`synonymMap`、`granularityRules` / `relationRules`，支持刷新与展开完整 JSON。

### API

- `GET /api/ontology/rules/status`
- `GET /api/ontology/rules/bundle`（规则中心数据）
- `POST /api/ontology/rules/reload`
- `POST /api/ontology/rules/import`（multipart 上传本体）
- `POST /api/files/<file_id>/ontology-validate`
