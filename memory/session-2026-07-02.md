## Completed Today

### 1. Backend ‚Äî Connection Extraction Performance (3 Bottleneck Fixes)

**Bug Fix ‚Äî `UnboundLocalError` in `_process_one_pack`:**
- Removed the `provider_call_success` event that was merged into `pack_start`.
- The `late_provider_response_ignored` nonlocal variable error was caused by a merge conflict between old and new code paths.

**Fix 1: Skip redundant candidate validation** (`mirror_kg_service.py`)
- Added `skip_granularity_validation: bool = False` parameter to `create_mirror_connection()`.
- Caller (`persist_connection_mirror_records`) passes `True` since candidates are already validated.
- Saves **2 DB lookups per connection** (~200 queries per 100 connections).

**Fix 2: Concurrent Mirror writes** (`llm_connection_extraction_service.py`)
- Refactored `persist_connection_mirror_records`: pre-compute all payloads (CPU-only), then write concurrently with `asyncio.Semaphore(5)`.
- Previous: sequential one-by-one (100 connections = 500+ sequential DB ops).
- Now: 5 concurrent write slots, ~3-5x faster.

**Fix 3: Merged per-pack event log calls**
- Combined `prompt_built` + `provider_call_start` ‚Üí single `pack_start` event.
- Removed `provider_call_success` (redundant ‚Äî parse result already proves success).
- Reduced per-pack DB writes from 4 to 2.

**Also: Added `PACK_CONCURRENCY` config** ‚Äî DeepSeek=4, Kimi=2, default=3 concurrent LLM calls.
**Also: Fixed** `created` vs `updated` counter ‚Äî previously `updated` was miscounted as `created`.

### 2. Backend ‚Äî Structured Progress API (`GET runs/{id}`)

Added new fields to `CompositeWorkflowRunRead`:
- `progress: ProgressDetail` ‚Äî total/completed/running/queued/succeeded/failed/skipped pack counts
- `stage_breakdown: StageBreakdownItem[]` ‚Äî per-stage pack counts, tokens, cost
- `connection_summary: ConnectionSummary` ‚Äî connection/projection counts
- `function_summary: FunctionSummary` ‚Äî function counts
- `usage_summary: UsageSummary` ‚Äî aggregated tokens, cost, elapsed time, estimated remaining
- `current_stage: str | None` ‚Äî currently active stage key
- `pack_concurrency: int` ‚Äî parallel pack count
- `execution_mode: str` ‚Äî "parallel" or "sequential"
- `recent_errors: RecentError[]` ‚Äî per-pack errors (capped at 10)

Helper `_build_progress_enrichment()` extracts all fields from existing `result_summary_json` and `step.response_json` ‚Äî **NO changes to execution logic**.
